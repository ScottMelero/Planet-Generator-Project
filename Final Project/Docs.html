Planet Generator

Scott Melero, Stmelero

Korbie Sevilla, Ksevilla

Goal

The goal of this project is to simulate a ‘universe’ where the user (who exists in a void of space) is capable of creating their own planet surface by interacting with RGB colored buttons/cubes within the sky.

The experience we are trying to create is interactive art. In essence we want the user to be able to create unique(random) worlds based on their interactions with buttons.

Obejcts and Interaction

The user will be interacting with different colored buttons/cubes in the sky. These buttons represent the type of worlds that can be generated on the plane. These interactions will be performed through clicks on the buttons. Once clicked there will be a HUD pop-up that provides an explanation of the world to be generated in accordance with the color. A clicked button will signify that the cube has been selected which will result in a unique world being generated on the plane the user in “standing” on.

Design & Style

The visual style of the world will be in between voxel/realistic. When it comes to visual references, we are aiming for something that applies a combination of minecraft texturing with more realistic texturing. The visual effects we will need to implement will consist of fog, collision detections, and canvas resizing to the entire screen. We are drawing design inspiration from games like no man's sky that feature procedurally generated planets and terrains. We are also looking at NASA’s global color mapping techniques where they can map a planet's topography based off of the surface color.

Advanced Techniques

HUD: There will be multiple HUD's that exist throughout this project. There is the "starting" HUD which will give directions on how to move/interact with the world. The other HUDs, which provide unique descriptions of the planets, are created when the user clicks the buttons.

Fog: This will be implemented for some of the worlds and is used to depict planets that have a more dense atmosphere. In essence, this is to strengthen the realistic perspective of our world.

Terrain Layering/Randomization: This technique is what will give each planet surface a unique/random surface of cubes. We are layering multiple blocks throughout the surface to create "jagged, natural" surfaces for a more realistic affect.

Color Picking: This will be accomplished through the buttons and help apply colors/textures to the surface terrain.

Miscellaneous features

Some features that will take some time include implementing textures for the various terrains we want to generate. Furthermore, it will be a process for us to place the objects in the correct places. For instance, we need to create three cubes(planets) and spread them accordingly across our sky. Doing such requires obtaining the correct vertices in accordance with the size of the map. Furthermore, we intend to use a ‘randomize’ feature that will help with creating unique terrains. Thus it will take some time to learn how to reference this function with the geometry of our world.

Note: In order to accomplish the above features we are using all libraries from the homework assignments as well as Javascripts Math.js library.
